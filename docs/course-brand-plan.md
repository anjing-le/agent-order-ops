# 七大 Agent 实战课课程品牌与教学物料规划

## 1. 整体课程定位

课程主名建议：

**同源七型 Agent 实战课**

副标题建议：

**一套工程脚手架，长出七种可运行业务 Agent**

这套课程面向 AI 应用开发者、后端/全栈工程师、产品技术负责人和希望沉淀 GitHub 作品集的学习者。课程重点不是反复学习底层技术栈，而是理解同一套工程底座如何面向不同业务场景生长出不同类型的 Agent。

所有项目都基于 `infra-dev-scaffolding`：

- 统一前后端技术栈。
- 统一目录结构和工程习惯。
- 统一 OpenAPI 契约和前端类型生成。
- 统一登录、权限、错误码、日志和质量检查。
- 统一 Git 提交、文档和演示交付方式。

课程核心主张：

- 不是概念课，而是可运行、可提交、可演示的完整工程课。
- 不是七套零散 Demo，而是一套脚手架生长出的七类 Agent 产品。
- 不把精力浪费在重复搭工程上，而是聚焦业务建模、Agent 编排、工具调用和人机边界。
- 每个项目都能形成 GitHub 仓库、课程讲义、演示路径和作品集素材。

一句话传播文案：

**不是学七套技术栈，而是用一套工程底座做七种 Agent。**

## 2. 七大 Agent 课程矩阵

| 仓库 | Agent 类型 | 主题 | 关键词 | 课程交付物 |
| --- | --- | --- | --- | --- |
| `agent-aigc` | 创作型 Agent | 全模态创作平台 | 多模型调度、Prompt 优化、图文音视频、异步生成、资产管理 | 创作工作台 |
| `agent-knowledge` | 知识型 Agent | RAG 智能知识库 | 文档解析、切片、Embedding、检索、引用、防幻觉 | 知识库问答系统 |
| `agent-customer-service` | 服务型 Agent | 智能电商客服 | 多轮对话、RAG、规则兜底、会话管理、转人工 | 客服会话台 |
| `agent-order-ops` | 执行型 Agent | 订单运营执行 | Tool Calling、审批、幂等、审计、失败补偿 | 订单执行控制台 |
| `agent-data-analyst` | 分析型 Agent | 数据分析工作台 | Text-to-SQL、指标口径、图表、洞察、安全查询 | 数据分析 Agent |
| `agent-finance-reconcile` | 审核型 Agent | 财务对账复核 | 票据解析、规则校验、异常复核、审计证据 | 对账审核台 |
| `agent-project-copilot` | 协作型 Agent | 项目交付协作 | 任务拆解、风险跟踪、会议纪要、周报复盘 | 项目协作 Copilot |

矩阵设计原则：

- 每个项目只讲一种典型 Agent 形态。
- 每个项目都保留同一套工程底座的痕迹，让学员形成迁移能力。
- 每个项目都有真实业务闭环，而不是单点功能演示。
- 每个项目都能单独作为 GitHub 作品，也能组合成完整课程矩阵。

## 3. 从入门到进阶的学习路径

### 阶段 0：工程底座认知

学习目标：

- 理解 `infra-dev-scaffolding` 的前后端结构。
- 理解 API 常量、OpenAPI、service boundary、质量脚本的作用。
- 理解为什么七个 Agent 不需要重复搭建基础工程。

推荐成果：

- 能独立启动一个基于脚手架的项目。
- 能看懂 README、`project_document`、`scripts` 和前后端目录。
- 能完成一次规范 commit 和 push。

### 阶段 1：知识与服务类 Agent

推荐顺序：

1. `agent-knowledge`
2. `agent-customer-service`

学习重点：

- RAG 基础链路。
- 多轮对话状态。
- 引用来源和防幻觉。
- 规则兜底和人工接管。

### 阶段 2：执行与审核类 Agent

推荐顺序：

1. `agent-order-ops`
2. `agent-finance-reconcile`

学习重点：

- Tool Calling 如何变成真实业务动作。
- 哪些动作必须审批。
- 如何设计幂等、审计日志和失败补偿。
- 如何让 Agent 的执行结果可追踪、可复核、可回滚。

### 阶段 3：创作与分析类 Agent

推荐顺序：

1. `agent-aigc`
2. `agent-data-analyst`

学习重点：

- 多模型能力编排。
- 异步生成任务。
- Text-to-SQL 的安全边界。
- 图表、洞察和结果解释。

### 阶段 4：综合协作类 Agent

推荐项目：

1. `agent-project-copilot`

学习重点：

- 会议纪要结构化。
- 任务拆解和责任人分配。
- 风险跟踪。
- 周报和复盘生成。

### 毕业交付

最终交付不是一份笔记，而是一组可展示作品：

- 7 个可运行 Agent 工程。
- 7 份项目 README。
- 7 条演示路径。
- 1 张课程总览图。
- 1 套 GitHub 作品集说明。

## 4. 视觉与品牌方向

### 品牌关键词

- 同源
- 工程化
- 业务现场
- 可运行
- 可提交
- 可演示

### 主视觉结构

推荐用“中心底座 + 七条分支”的结构：

- 中心：`infra-dev-scaffolding`
- 分支：七类 Agent
- 末端：七个业务工作台

这种结构能直接表达课程核心：所有 Agent 都从同一套脚手架生长出来，而不是彼此孤立。

### 视觉元素

适合使用：

- 工程模块卡片
- API 契约线
- Agent 工具节点
- 业务看板截图
- Git 提交链路
- 课程路线图

避免使用：

- 只有抽象光效的 AI 背景
- 过度赛博风装饰
- 与业务无关的概念插画
- 让人误以为是纯理论课的海报

### 色彩建议

底座色：

- 深灰
- 蓝灰
- 低饱和科技蓝

项目识别色：

- 创作型：亮蓝或青色
- 知识型：绿色
- 服务型：橙色
- 执行型：红色或玫红
- 分析型：紫色
- 审核型：金色或琥珀色
- 协作型：靛蓝或蓝紫

整体原则：

- 保持工程感。
- 七个项目能区分，但不要割裂。
- 海报可以更有冲击力，课程讲义和 GitHub README 要更克制。

## 6. 七个单项目物料文案

### 6.1 agent-aigc：创作型 Agent

海报主文案：

**从 Prompt 到资产库，搭一个真正可用的全模态创作 Agent。**

视频封面文案：

**创作型 Agent：多模型调度与异步生成**

教学卖点：

- 如何把文本、图片、音频、视频能力组织成统一创作平台。
- 如何设计 Prompt 优化、模型选择和生成参数。
- 如何用异步任务管理生成过程。
- 如何沉淀素材资产、生成历史和可复用模板。

演示路径：

1. 创建创作任务。
2. 选择模型和生成参数。
3. 查看异步生成状态。
4. 管理生成资产。
5. 复用 Prompt 模板。

### 6.2 agent-knowledge：知识型 Agent

海报主文案：

**让知识库回答有出处，让 Agent 不再凭空发挥。**

视频封面文案：

**知识型 Agent：RAG、引用与防幻觉**

教学卖点：

- 文档上传、解析和切片策略。
- Embedding 与检索召回。
- 答案引用来源展示。
- 防幻觉提示和低置信度处理。

演示路径：

1. 上传知识文档。
2. 查看解析和切片结果。
3. 发起知识问答。
4. 查看引用来源。
5. 对低置信度问题触发兜底。

### 6.3 agent-customer-service：服务型 Agent

海报主文案：

**从一句咨询，到一次完整的客服闭环。**

视频封面文案：

**服务型 Agent：多轮会话、规则兜底、转人工**

教学卖点：

- 多轮会话状态如何设计。
- RAG 与业务规则如何配合。
- 售前、售后、物流、退款等意图如何识别。
- 什么时候转人工，转人工前要交接什么上下文。

演示路径：

1. 用户发起咨询。
2. Agent 识别意图并召回知识。
3. 多轮追问补齐信息。
4. 命中规则兜底。
5. 转人工并携带会话摘要。

### 6.4 agent-order-ops：执行型 Agent

海报主文案：

**让 Agent 不只会回答，还能安全地执行订单动作。**

视频封面文案：

**执行型 Agent：Tool Calling、审批与审计**

教学卖点：

- Tool Calling 如何映射到订单业务动作。
- 如何设计执行计划和人工确认边界。
- 如何处理幂等、审计日志和失败补偿。
- 如何让 Agent 执行动作可追踪、可解释、可复核。

演示路径：

1. 查询订单上下文。
2. 生成 Agent 执行计划。
3. 执行发货拦截、地址修改、退款或优惠补偿。
4. 高风险动作进入审批。
5. 查看审计日志和补偿记录。

### 6.5 agent-data-analyst：分析型 Agent

海报主文案：

**把自然语言问题，变成安全可解释的数据洞察。**

视频封面文案：

**分析型 Agent：Text-to-SQL 与指标分析**

教学卖点：

- 指标口径和数据字典如何约束问题理解。
- Text-to-SQL 如何生成、校验和执行。
- 如何阻断危险查询。
- 如何从表格结果生成图表和洞察摘要。

演示路径：

1. 输入业务分析问题。
2. Agent 识别指标和维度。
3. 生成 SQL 并进行安全校验。
4. 展示数据表和图表。
5. 生成洞察解释。

### 6.6 agent-finance-reconcile：审核型 Agent

海报主文案：

**让每一笔异常都有规则、有证据、有复核路径。**

视频封面文案：

**审核型 Agent：票据解析与对账复核**

教学卖点：

- 票据字段抽取和结构化。
- 对账规则如何表达。
- 异常如何分级和解释。
- 审计证据如何留存。

演示路径：

1. 上传票据或对账数据。
2. Agent 解析关键字段。
3. 执行规则校验。
4. 标记异常并给出原因。
5. 生成人工复核证据。

### 6.7 agent-project-copilot：协作型 Agent

海报主文案：

**把会议、任务、风险和周报，串成项目交付助手。**

视频封面文案：

**协作型 Agent：任务拆解与项目复盘**

教学卖点：

- 会议纪要如何结构化。
- 任务如何拆解到负责人和截止时间。
- 风险如何持续跟踪。
- 周报和复盘如何基于真实项目状态生成。

演示路径：

1. 输入会议纪要。
2. Agent 抽取任务和风险。
3. 生成项目看板。
4. 跟踪延期和阻塞项。
5. 生成周报和复盘摘要。

## 8. 单项目课程拆分模板

每个项目建议固定为 7 节：

1. 业务场景拆解：用户是谁，Agent 要解决什么问题。
2. Agent 能力边界：能自动做什么，必须人工确认什么。
3. 数据模型与接口：核心实体、API、OpenAPI 契约。
4. 后端实现：服务编排、工具调用、状态流转。
5. 前端实现：工作台页面、操作路径、结果展示。
6. 质量验收：测试、构建、脚本检查、演示脚本。
7. Git 收口：提交、推送、README、截图与复盘。
